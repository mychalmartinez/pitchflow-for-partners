# Pitchflow CMS Schema Build

Production schema build prompt for the `webflow-migration-collection-builder` skill. Builds the four CMS collections that back the Pitchflow for Partners pitch page system.

---

## Schema source and audit trail

This prompt is derived from two sources:

1. `docs/canonical-reference.md`, the project's source of truth for page structure, CMS schema, and voice rules.
2. The locked v5 Pitchflow template (the brand-agnostic React reference component that every pitch page hydrates against).

The schema has been audited against both. Field types, required flags, option choices, reference cardinality, slugs, and help text all reflect the locked template and canonical schema spec.

### One deliberate addition beyond the canonical Field Summary

The canonical CMS Field Summary for Pitch Pages lists 35 custom fields. This prompt adds a 36th: `Document Title`. Rationale: during template hardening, the navigation document title was moved from a hard-coded JSX string into a CMS-bound field so the binding to Webflow Designer is explicit at template-rebuild time. Storing the generated string in the CMS keeps the orchestrator skill as the single source of truth for that string and avoids string-templating logic inside the Designer. The field is PlainText, required, populated by the orchestrator at item creation, editable by the PAM during review.

### Deliberate omissions from the canonical SEO Spec

The canonical reference's SEO and Meta Field Spec section lists seven SEO-related fields. This schema includes only the three required ones (Meta Title, Meta Description, Open Graph Image). The four optional fields are intentionally omitted:

- **Open Graph title** and **Open Graph description**: per canonical they fall back to Meta Title and Meta Description respectively. The fallback can be configured at the Webflow Collection Template page level (binding the OG fields to the Meta fields directly) without needing separate CMS columns.
- **Twitter card type**: a Webflow page-level SEO setting, not a per-item field.
- **Canonical URL**: auto-generated by Webflow from the page slug.

If per-item override capability for OG Title or OG Description is wanted later, adding them is a non-breaking schema migration. They are not blocking for the orchestrator skill build.

### Fields

| Field name | Type | Required | Help text |
| :--- | :--- | :--- | :--- |
| Title | PlainText | Yes | Executive's role at the brand (for example, "Chief Technology Officer", "Head of Growth"). |

| Role Type | Option | Yes | Executive role type. Determines which mandate framing applies. CTO required where the technology decision is in scope. |

### Option field choices

**Role Type:** CEO, CTO, CMO, Growth, Brand, Content, Digital, Other

## Collection 2. Capability Cards

| Property | Value |
| :--- | :--- |
| Display name | Capability Cards |
| Singular name | Capability Card |
| Slug | `capability-cards` |
| Public Collection page | No |
| Built-in Name field stores | Internal CMS label for the card (typically the same as Title, but may differ when one Title is reused across multiple brand-specific cards) |

**Important:** Name (built-in) and Title (custom field below) are intentionally separate per the canonical reference. Name is the CMS-internal label content editors see in lists. Title is what displays on the page. They may be identical or differ depending on whether a card title needs to be disambiguated for CMS browsing.

### Fields

| Field name | Type | Required | Help text |
| :--- | :--- | :--- | :--- |

| Priority Tag | PlainText | Yes | Brand priority pulled from the brand's stated 2026 goals. Displayed as a small tag below the title. |
| Today | RichText | Yes | Current-state framing paragraph naming the gap this card addresses, in the brand's specific terms. |
| With Agency | RichText | Yes | Capability paragraph. Must open with "With [Agency Name] and Webflow, we…". Names what the agency-platform partnership delivers. |
| Bullet 1 | PlainText | Yes | First outcome bullet. Brand-specific, tied to the brand's 2026 priorities. |
| Bullet 2 | PlainText | Yes | Second outcome bullet. Brand-specific, tied to the brand's 2026 priorities. |
| Bullet 3 | PlainText | Yes | Third outcome bullet. Brand-specific, tied to the brand's 2026 priorities. |
| Order | Number | Yes | Display order in the capability cards grid. Use 1, 2, or 3. |
| Service Line | Option | Yes | Which of the five capability service lines this card represents. |
| Feature 1 | PlainText | Yes | First Webflow Enterprise feature pill. Use canonical product names from webflow.com/llms.txt exactly (for example, "CMS", "Optimize"). |
| Feature 2 | PlainText | Yes | Second Webflow Enterprise feature pill. Canonical product name. |
| Feature 3 | PlainText | No | Third Webflow Enterprise feature pill. Optional. Canonical product name. |

### Option field choices
